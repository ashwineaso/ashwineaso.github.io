Internal Content Management Application

The company

Avanti Learning Centers provides after-school STEM tutoring to 6000+ high school students in 40 locations across India.

The users

40+ dedicated employees who create content for in-class materials

Purpose of this product

Users could add finalized content, then generate automatically formatted materials.

My role

First full-time UX Designer in the company

Other team members

Product Manager, Lead Developer

Duration

3 months (in 2015)

Design Changes

Navigation redesign

I improved the application by designing a consistent header with meaningful labels. I reorganized the content structure to match the user's mental model and added a menu to access the new categories of content.

Visual equation editor feature

Also, users now have an option to type equations in the format of their choice using this embedded equation editor.

Process

My brief

The Product Manager told me that users were unsatisfied with the existing application. My brief was to identify the problems and propose design solutions.

Conducted interviews

The company did not have a formal onboarding process or documentation on the user's workflow. So, I conducted multiple interviews to understand the problems in detail.

Identified problems in the existing application

During the interviews, users talked about their frustrations with the existing application.

Uncovered opportunities for improvement

The content creation process involved 4 users working collaboratively through multiple revision and approval cycles. They managed their process using Dropbox, MS Excel, and MS Word. Because their workflow was split between these disconnected tools, they had to juggle between 14+ file versions and endless comments and emails. The rich interview data inspired me to think about how we could support this collaborative process in a centralized location in the application.

Documented design ideas

I sketched early concepts and documented my ideas in the form of wireframes.

Prioritized changes

Before evolving my ideas further, I talked to the product manager to understand the technical constraints. Based on the availability of resources, we prioritized changes that could be implemented in the next release.

Created mockups and got feedback

I created high-fidelity mockups for prioritized changes and showed them to the Content Team Lead, the company Founder, the Product Manager, and the Lead Developer to get their feedback.

Iteration 1 inspired by existing file structure
Iteration 2 based on user feedback

Conducted card-sorting

In the existing application, categories that belonged together were grouped separately, and labeling was different from the user's terminology. So, I conducted a card-sorting exercise to find the user's mental model of content categorization.

Implemented and tested with 2 users

In the face of resource constraints, I took the initiative to code the HTML and CSS for the new design. After we implemented all the changes, I tested the application remotely with two users using Steve Krug's think-aloud method.

Impact

I found that in the new design, the users were able to find content without getting confused or lost at any step. We also learned new insights that could inspire improvements in the future.

Reflection

Using the user-centered design process, we were able to improve the product, help users do their job better, and ultimately move the company forward. Throughout the process, I maintained constant communication within the team and created a design that balanced user needs and technical constraints. By involving the Product Manager and the Lead Developer in the user testing phase, I introduced a culture of talking to users during design and implementation.

If I were to do this project again, I would measure time on task and user satisfaction before and after the redesign.

Future iterations

One of the users said that "Every time we have to add something in a topic, we have to click on the chapter name, then topic name, then find what we need. It's frustrating to do this repeatedly (several times a day)." To solve this problem, I would include a feature to access recently visited content from the landing page.

Also, I would further explore and test features to enable users to work collaboratively to create content. A centralized real-time content editing tool could save them the manual task of emailing changes and managing multiple file versions.
